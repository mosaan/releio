本フォルダは「UMLによる統一ソフトウェア開発プロセス」の書籍の内容全文、およびその要点を格納しています。

* `FULLTEXT`フォルダ：書籍の原文をそのまま格納しています。リファレンスとして必要に応じて参照してください(※著作権の観点から、原著はGitに格納されません。)
* `統一プロセス：01_構成単位と要素.md`: プロセスを構成する主要な構成単位を、上位概念から下位概念へとブレークダウンした形でまとめた資料です。
* `統一プロセス：02_成果物と責任ワーカー.md`: プロセスの主要な基本ワークフロー（要求、分析、設計、実装、テスト）ごとに、そこで作成される主な成果物と、その成果物に対してを負うワーカー（Worker）まとめた資料です。
* `統一プロセス：03_ワークフローとフェーズ別活動の要点.md`: プロセスにおける主要な基本ワークフローごとの「作業」（アクティビティ）と、各フェーズ（方向づけ、推敲、作成、移行）においてそれらの作業がどのように重点的に実施されるかをまとめた資料です。

構成要素の全体像を把握するには`統一プロセス：01_構成単位と要素.md`を、「今のフェーズ、ワークフローで次に作業として何をすればいいか」は`統一プロセス：03_ワークフローとフェーズ別活動の要点.md`を参照するとよいでしょう。現状の成果物の整理状況を俯瞰するには`統一プロセス：02_成果物と責任ワーカー.md`が便利です。これらの文書はあくまで概要を示したものであるため、具体的な作業を進めるうえでは`FULLTEXT`フォルダ内の文書を必ず確認する必要があります。特に、具体的な作業内容の定義はほとんどが `docs\UP_with_UML\FULLTEXT\UMLによる統一ソフトウェア開発プロセス_第2部.md` に含まれます。`mdatlas`ツールを用いて文書の構造を把握したうえで、適切な章を参照しながら作業を進めなさい。

書籍で示されるプロセスの実施状況を管理するために、あなたは`.agent/UP_PROGRESS.md`というファイルを用います。これは以下のような内容で構成されます:

```markdown
# 反復 N(数値) の作業状況

- 現在のフェーズ: [方向づけ/推敲/作成/移行]
- 現在のワークフロー: [要求/分析/設計/実装/テスト]
  - 現在の作業: [...]

## 反復 N-1 までのサマリ

[過去の反復までの作業状況を端的に示すパラグラフをここに記載します。初めての反復(N-1 = 0)の場合は、その旨を記載します。]

## 反復 N における主要な目標

[現在の反復における主要な目標を簡潔に記載します。]

# 反復 N+1 における目標の候補

[現在の反復において発見された要件等・調査事項などを記録し、次の反復において何に取り組むべきかを明らかにします。これは優先順位付けられたリストのような形態になりえます。]
```

また、統一ソフトウェア開発プロセスにおける作業成果物を`docs_UP/`フォルダ配下にまとめることとします。
基本的には`成果物名.md`のMarkdownファイル形式を用いますが、個々の成果物について分割が必要である場合は`成果物名_分割された分類名.md`のように、アンダースコアで再分類を区切って示すスタイルをとることとします。
UMLを用いた図をMarkdown中で使用する必要がある場合、その記述にはPlantUMLを用いることとします。PlantUMLを用いた図のテキスト表現はコードブロック(```plantuml...```)を用いて記述する必要があります。
